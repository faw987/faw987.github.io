Then open http://localhost:8080/windates.html
Everything else (Save/Load, events, alarms) works fine from file://.
How to get an OAuth Client ID (step-by-step)

Important: Multiple Google Accounts

If you have more than one Google account (e.g. a personal account and a work account), pay attention to which account you are using at each step. There are two separate choices:

  • Google Cloud Console account — the account where you create the OAuth Client ID (Step 1 below). This account owns the "project" and API credentials. It does NOT have to be the same account that stores the spreadsheet.
  • Google Drive account — the account you sign into when WebDates asks to authorize. This is where the spreadsheet gets created and stored. You can choose any of your accounts at that point.

Tip: Before starting, open an incognito/private window or make sure you're signed into the Google account you want to use for the Cloud Console. Look at the avatar in the top-right corner of the Google Cloud Console to confirm which account is active.

Step 1 — Go to Google Cloud Console

Open console.cloud.google.com and sign in. If you see an account picker, choose the account you want to own this project (any of your accounts will work).

Step 2 — Create a new project (or use an existing one)

  • Click the project dropdown at the top of the page (it may say "Select a project" or show a project name).
  • Click New Project.
  • Name it anything you like (e.g. "WebDates").
  • Click Create. Wait a few seconds, then select the new project from the dropdown.

Step 3 — Enable the APIs

  • In the left sidebar, go to APIs & Services → Library.
  • Search for Google Sheets API. Click it, then click Enable.
  • Go back to the Library. Search for Google Drive API. Click it, then click Enable.

Step 4 — Configure the OAuth consent screen

  • In the left sidebar, go to APIs & Services → OAuth consent screen.
  • Choose External user type, then click Create.
  • Fill in the required fields:
    • App name: WebDates (or anything)
    • User support email: select your email
    • Developer contact email: your email
  • Click Save and Continue through the remaining screens (Scopes, Test users, Summary). You can skip adding scopes here — WebDates requests them at runtime.

Step 5 — Add yourself as a test user

  • On the OAuth consent screen page, scroll down to Test users.
  • Click Add users.
  • Enter the email address(es) of every Google account you want to use with WebDates. For example, if you have you9422@gmail.com and you987@gmail.com, add both here.
  • Click Save.
  • Why? While the app is in "Testing" mode (not published), only listed test users can sign in. This is normal and fine for personal use.

Step 6 — Create an OAuth Client ID

  • Go to APIs & Services → Credentials.
  • Click + Create Credentials → OAuth client ID.
  • Application type: Web application.
  • Name: WebDates (or anything).
  • Under Authorized JavaScript origins, click + Add URI and enter:
    http://localhost:8080
    If you use a different port, add that too (e.g. http://localhost:8081). If you deploy to a real domain later, add that URL as well.
  • Leave Authorized redirect URIs empty (not needed for this flow).
  • Click Create.

Step 7 — Copy the Client ID

  • A dialog shows your Client ID and Client Secret. You only need the Client ID.
  • It looks like: 123456789-abcdef.apps.googleusercontent.com
  • Copy it and paste it into the OAuth Client ID field above.
  • (You do NOT need the Client Secret for this app.)

Step 8 — Test it

  • Check the Enable Google Sheets sync checkbox above.
  • Paste your Client ID into the field.
  • Click Sheets Save in the toolbar.
  • A Google sign-in popup appears. If you have multiple accounts, it will show all of them — pick whichever account you want the spreadsheet stored in. This does not have to be the same account you used for the Cloud Console.
  • You may see a "This app isn't verified" warning — click Advanced → Go to WebDates (unsafe). This is normal for testing-mode apps.
  • Grant the requested permissions (Sheets + Drive access for files this app creates).
  • Your calendar data will be saved to a spreadsheet in that account's Google Drive!

Summary: Multiple Accounts

Step Which account?
Cloud Console (create project + Client ID) Any account. Pick one and stick with it for the Console.
Test Users list Add ALL accounts you plan to sign in with.
Sign-in popup (when you click Sheets Save/Load) Choose whichever account should own the spreadsheet.

The Client ID is the same regardless of which account signs in. You create it once and can use it with any Google account that's listed as a test user.
